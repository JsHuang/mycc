# Link Analyzer - PoC编写分析器

## 核心目标

只回答一个问题：**这个链接是否包含足够详细的技术信息，使您能基于此编写出一个可运行的漏洞利用代码（PoC）？**

## 设计理念

- **目标聚焦**：只关注"能否编写PoC"
- **增强判断**：结合NVD官方描述，判断链接是否提供**超出官方描述**的技术细节
- **移除干扰**：不分析漏洞检测、加固、攻击事件、威胁情报等
- **输出详实**：展示分析依据，支持可追溯

---

## 判断标准（需满足至少2项）

### ✅ 有效技术信息

1. **具体执行代码**：可直接运行的代码片段
   - ✅ `requests.post(url, data={'cmd': '; whoami'})`
   - ❌ "攻击者可以执行任意代码"

2. **具体触发参数**：精确的值或格式
   - ✅ "Content-Length > 0x7fffffff 时触发"
   - ❌ "发送恶意请求"

3. **完整攻击流程**：可复现的步骤序列
   - ✅ "1. POST /api/login 2. 设置 Authorization 头 3. 访问 /admin"
   - ❌ "通过认证绕过获取权限"

4. **具体漏洞位置**：精确的代码位置
   - ✅ "`parseXML()` 函数第127行"
   - ❌ "认证模块存在漏洞"

5. **原创分析过程**：调试、逆向、fuzzing过程
   - ✅ "fuzzing发现`\x00`导致崩溃"
   - ❌ "根据官方描述，这是缓冲区溢出"

## 使用示例

### 示例1：可编写PoC（有CVE）
```
CVE编号：CVE-2026-20127

NVD官方描述：
认证绕过漏洞，允许未认证远程攻击者绕过认证并获得管理员权限。

链接内容对比：
除了官方描述外，还提供了具体的HTTP请求示例、触发参数和代码执行流程。

✅ 是否可编写PoC：是
原因：提供具体Python代码片段（requests.post调用）和触发参数（Content-Length > 0x7fffffff），满足两项标准。
```

### 示例2：不可编写PoC（无CVE）
```
CVE编号：未提及

NVD官方描述：
无

链接内容对比：
仅描述漏洞影响，无具体技术细节。

✅ 是否可编写PoC：否
原因：仅描述漏洞影响为RCE，未提供任何具体代码、参数或复现步骤。
```

### 示例3：重复官方描述（不可编写）
```
CVE编号：CVE-2022-20775

NVD官方描述：
CLI命令访问控制不当，允许已认证本地攻击者提升权限至root。

链接内容对比：
内容完全重复NVD官方描述，未提供额外技术细节。

✅ 是否可编写PoC：否
原因：内容与官方描述一致，无额外代码、参数或复现步骤。
```

## 与原版的区别

| 维度 | 原版 | 改进版 |
|------|------|--------|
| **目标** | 只判断能否写PoC | 判断是否**超越**官方描述 |
| **分析深度** | 基于内容本身 | 结合NVD官方描述对比 |
| **输出** | 一句话 | 5部分结构化输出 |
| **可信度** | 一般 | 高（有官方基准） |

---

## 不关心的内容

以下内容即使存在也**不影响判断**：
- 漏洞检测方法
- 安全加固建议
- 攻击事件报告
- 威胁情报/IOC
- 网络配置建议
- 补丁版本信息
- CVSS评分
- YARA/Sigma规则

**只关心：能否写PoC？**


---

## 重要提示

- **保守判断**：不确定时回答"否"
- **不生成PoC**：仅分析可行性，不提供代码
- **不评估风险**：只评估技术可行性，不评估危害性
- **语言无关**：中英文均可，只要技术信息明确
- **NVD优先**：当链接内容与NVD官方描述一致时，回答"否"
