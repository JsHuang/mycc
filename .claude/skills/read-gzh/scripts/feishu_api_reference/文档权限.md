下面给你一份**最实用、直接可抄**的
飞书开放平台「docs / docx / drive 权限清单 + 最小权限组合」，开发直接用。

# 一、一句话速记（必背）
- **docx** = 新版云文档**内容读写**
- **docs** = 旧版云文档**兼容**
- **drive** = 云盘**文件/文件夹/位置/上传下载**

> 新项目**只需要：docx + drive**，docs 可不用。

---

# 二、完整权限清单（官方标准）

## 1）docx（新版文档）
- **docx:document:readonly**
  只读：获取文档内容、元信息、块、评论
- **docx:document**
  读写：创建、编辑、删除、权限管理、版本管理
- **docx:document:create**
  仅创建文档
- **docx:document:block:convert**
  块转换（很少用）

## 2）docs（旧版文档）
- **docs:doc:readonly**
  旧文档只读
- **docs:doc**
  旧文档读写+管理

## 3）drive（云盘/文件/文件夹）
- **drive:drive:readonly**
  查看云盘、文件夹、文件列表
- **drive:drive**
  管理云盘结构：创建/移动/重命名/删除文件夹
- **drive:file:readonly**
  查看文件元信息
- **drive:file**
  文件全操作：上传、下载、复制、预览、删除
- **drive:file:upload**
  仅上传
- **drive:file:download**
  仅下载
- **drive:permission**
  管理文件/文件夹权限（添加协作者等）

# 三、最常用「最小权限组合」直接复制

## 场景1：只读取所有文档内容（最常用）
```
docx:document:readonly
drive:drive:readonly
```

## 场景2：读取 + 创建/编辑文档内容
```
docx:document
drive:drive:readonly
```

## 场景3：完整文档管理（读写+移动+上传）
```
docx:document
drive:drive
drive:file
```

## 场景4：只做文件上传/下载
```
drive:drive:readonly
drive:file:upload
drive:file:download
```

## 场景5：管理权限（给别人加权限）
```
drive:permission
```

## 场景6：兼容旧文档（老系统才需要）
```
docs:doc:readonly
docs:doc
```
